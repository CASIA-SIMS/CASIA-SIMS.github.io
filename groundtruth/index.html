Groundtruth

The timestamped translation vector and unit quaternion of the ground truth trajectory (from the camera coordinate to the world coordinate. Format:

timestamp[i] p_x p_y p_z q_x q_y q_z q_w

The ground truth 3D point cloud models are acquired simultaneously with the corresponding trajectories using GeoSLAM. In the same scene, for different trajectories, the models are in different coordinate systems. Using CloudCompare2, we merge the 3D point cloud models as a complete model for each scene. The process involves a first coarse alignment by manually selecting point correspondences and a densely sampled fine alignment using Iterative Closest Point (ICP). Based on the two steps, a global calibration for each scene is given so that the 3D point cloud models are unified in a global coordinate system. After manually eliminating moving objects, we provide a complete point model with the standard .las file format for each scene. Similarly, transforming trajectories from the original world frame to the global coordinate frame is done by applying the rigid transformation.
